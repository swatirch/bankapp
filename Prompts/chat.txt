We start today with Phase 1. Let's set up the project.
Step 1 — Project Structure (the why)
We'll use Maven because it's the industry standard for Java dependency management. Here's the structure we're building toward:
banking-app/
├── pom.xml
├── src/
│   ├── main/
│   │   └── java/
│   │       └── com/banking/
│   │           ├── domain/        ← Pure business logic (no Spring here)
│   │           │   ├── Account.java
│   │           │   └── Transaction.java
│   │           ├── service/       ← Use cases (what the app does)
│   │           │   └── AccountService.java
│   │           └── repository/    ← Data access (interface only for now)
│   │               └── AccountRepository.java
│   └── test/
│       └── java/
│           └── com/banking/
│               └── service/
│                   └── AccountServiceTest.java  ← We write THIS first
Why this structure?
domain/ has zero dependencies — it's pure Java. This is your business heart. It should never care if you're using Spring, Kafka, or MySQL.
service/ orchestrates domain objects — it contains your use cases
repository/ is an interface — the service doesn't know or care if data lives in MySQL, MongoDB, or memory. This is the Dependency Inversion Principle
