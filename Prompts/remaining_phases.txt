KEY DESIGN DECISIONS (never change these)
═══════════════════════════════════════════════════════
- BigDecimal for money, never double
- No setters on domain objects
- Entity/Domain/DTO separation
- H2 for tests, PostgreSQL for dev
- Flyway owns schema, never edit existing migrations
- JWT stateless auth
- TDD always: red → green → refactor
- Pain first: feel the problem before the solution
- AWS for cloud (Phase 11)
- Spring Authorization Server for OAuth 2.0 (Phase 9)
- Guard clauses pattern (fail fast, happy path last)
